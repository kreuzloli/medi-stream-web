# 内容页面与证书查询设计

## 目标

依据 `/Users/iris/Documents/live/ui/` 中的美工图，在现有 `medi-stream-web` 中补齐精选专题、科研培训和证书查询相关页面，并从 Home 页面建立完整跳转链路。实现保持现有 Vite、TypeScript、原生 Web Components 和 Hash Router 技术栈，不新增依赖。

## 页面范围

- 精选专题列表：`#/topics`
- 科研培训列表：`#/training`
- 科研培训详情：`#/training-detail?id=<id>`
- 证书查询表单：`#/certificates`
- 证书详情：`#/certificate-detail?id=<id>`

列表页和详情页都复用现有 `<medi-header>`、轮播组件和 `<medi-footer>`。Header 保持 Home 当前实现，不增加美工图最右侧的浏览记录、收藏、文件夹和日历四个图标。

## 页面结构

### 精选专题列表

轮播图下展示“精选专题”标题和五列专题卡片网格。卡片包含封面、标题、更新时间、关注状态和三条子主题。点击卡片进入专题内容时，当前阶段以无后端依赖的可点击状态呈现，不扩展美工图未提供的专题详情页。

### 科研培训列表与详情

列表页展示轮播、“科研培训”标题、证书查询入口和纵向培训条目。每条包含封面、标题、日期和“查看详情”按钮，点击后携带培训 ID 进入详情页。

详情页根据 Hash 查询参数加载内容，展示轮播、栏目标题、证书查询入口，以及白底文章区。文章区包含标题、日期、来源、正文、内容图片和相关链接。

### 证书查询与详情

查询页展示姓名、身份证号码、手机号码三个输入项。提交前要求三项非空；校验失败时在表单内显示明确提示。提交后调用查询 service，成功则携带证书 ID 跳转详情页，未查到或请求失败则显示错误信息。

详情页读取证书 ID 并调用详情 service，展示基础信息和证书信息表格。加载、失败、无数据都有页面内状态，不吞掉异常。

## 数据与 API

新增内容 service，预留以下临时路径：

- `GET /topics`
- `GET /trainings`
- `GET /trainings/:id`
- `POST /certificates/query`
- `GET /certificates/:id`

service 仍通过现有 `buildApiUrl` 和认证 Header 发起请求。由于后端路径尚未确定，页面捕获请求失败后使用本地假数据继续展示；日志记录页面类型、业务 ID 和失败原因。未来接入真实接口时只需调整路径和响应归一化，不改页面组件契约。

## Home 与导航跳转

- Header 使用 Hash Router 跳转，并根据当前路径显示激活项。
- Home 精选专题区的“更多”和卡片跳转至 `#/topics`。
- Home 科研培训相关入口跳转至 `#/training`；若 Home 当前没有独立培训区，则 Header 的“科研培训”作为主要入口，不额外创造美工图之外的首页区块。
- Header 的“证书查询”跳转至 `#/certificates`。

## 视觉与响应式

桌面端以 1920 像素美工图为基准，沿用现有 Home 的内容宽度、蓝色品牌色、浅灰页面背景、轮播和 Footer。页面组件使用共享内容页样式，避免复制 Header、轮播和 Footer 样式。

窄屏下导航允许横向滚动或紧凑排列，专题网格逐级减少列数，培训条目改为上下布局，表单和证书表格保持可读并避免横向溢出。

## 兼容性与边界

- 不修改公共 API、后端接口、数据库结构或现有 Home 数据模型。
- 不引入新依赖，不重写现有路由器。
- 不实现美工图右侧四个 Header 图标。
- 不实现美工图未提供的专题详情、科普视频和关于我们页面。
- 假数据只作为接口不可用时的回退，页面调用链保持真实异步请求结构。
